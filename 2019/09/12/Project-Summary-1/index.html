Music Genre Classification with Machine Learning

Introduction

The problem our team tackled is to predict which genre would a random piece of music fall into, such as metal, classical, hiphop, blues, pop, and reggae. We employed Python, Pandas, and NumPy to write our codes that teach our machine how to identify genre of the chosen music piece. We have collections of different music pieces classified by their music genre. This is supervised machine learning since it has both the features (data of sound) and the targets (genre). We used kNN(k-nearest neighbor), SVM(support vector machine), random forest, and neural network approaches to train our algorithm how to use the existing data for future prediction.

Data Acquisition

Sound waves form due to the vibration of air molecules. The external ear captures the sound waves, which are then sent to the middle ear. The eardrum then vibrates back and forth, converting sound into signals later transmitted to the brain via auditory nerves. The diagram below presents a more direct way of showing how the ear works to perceive sounds. Computers, unlike humans, use a totally different way of capturing sound. Computers could only express digital signals, which is in binary form and could not take fractions, but sound waves are all in analog signals, which are continuous waves in sine wave form and could take any value of amplitude. To attain our data, we first need to convert analog signals into digital signals using the Fourier transformation to let the computer identify and read the values in its ordinary way.


Data Preparation

Our team imported Python libraries such as pandas and NumPy to convert sound files to numeric data which computer can recognize. The first step was to use pandas to determine the number of rows and number of columns of the dataset with features for each column. For instance, this procedure was performed using five data points with different features to create a dataset with five rows and thirty seven columns with name of genre as the last column. After obtaining an entire dataset, our group splited it into two separate sets: train dataset and test dataset. The next step was to convert pandas.DataFrames into NumPy Arrays as to creat a grid of values of the same type and to have a tuple of integers giving the size of array along each dimension.


Train Model

Our team used the tool knn.fit and set the value of k in the range of one to eleven in order to find the optimal value for k. We picked k that produces the maximum accuracy and use it as our optimal k for this training model and then tested the results using knn.predict. Because we knew the genres of the music pieces used for testing, we could compare them against the predictions the model made.

Determine Performance Matrix

Confusion matrix was used to observe exact numbers of times of correct and incorrect predictions.

Interpret Model and Report Results

The test score for KNN is 0.671 with a training score of 0.789; the test score for Random Forest is 0.653 with a training score of 1.000; the test score for SVM is 0.700 with a training score of 1.000; furthermore, the test score for Neural Network is 0.676 with a training score of 1.000. Because the SVM algorithm, an algorithm that performs the best in classifying two music genres, has the highest test score among all four algorithms, which is 0.700, and has the highest training score of 1.000, it is the optimal estimator. Since it has uniform occurrence of correctly identifying all genres of music, shown in its confusion matrix, it is the most appropriate algorithm to use for identifying music genre.

Conclusions

Although the model we built is capable of correctly identifying the genre of music, there were instances in which it made errors in identification. To mitigate this error to minimum, we need more data for each genre to help the machine distinguish two pieces of music that are similar in sound but are indeed different in genre. We could also change the proportion of data in the training set and the proportion of data in the testing set, i.e. increasing the number of data points used for training. Overall, with the help of different types of libraries, we could sort the data more efficiently with clearer boundaries to help the machine learn faster. For now, our machine performs optimally, and it will perform even better with further modifications.
